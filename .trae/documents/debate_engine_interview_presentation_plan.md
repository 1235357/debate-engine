# DebateEngine 项目面试讲解计划

## 1. 项目概述

### 1.1 项目简介
- **项目名称**：DebateEngine
- **核心定位**：结构化多智能体批判与共识引擎
- **解决的问题**：将自由文本 AI 投票升级为结构化交叉批判循环，解决现有多智能体系统的局限性
- **价值主张**：提高 AI 决策的质量、可靠性和可审计性

### 1.2 核心功能
- **结构化批判**：使用 Pydantic v2 CritiqueSchema 约束输出，确保一致性和可解析性
- **多智能体交互**：支持单轮批判和多轮辩论，包含提案、交叉批判、匿名化、法定人数检查和法官总结
- **反从众机制**：三层反从众防御（提供商多样性法定人数、魔鬼代言人、响应匿名化）
- **评估指标**：7 个全面的 DebateEval 指标，包括原创的 CIS 指标
- **集成能力**：支持 Python API、FastAPI REST、MCP Server 和 GitHub Action 集成
- **免费API策略**：完全基于免费API提供商运行，降低使用成本

## 2. 技术架构

### 2.1 分层架构
- **入口层**：Python API、FastAPI REST、MCP Server
- **编排层**：QuickCritiqueEngine（单轮批判）、DebateOrchestrator（多轮辩论）
- **提供商层**：LiteLLM（支持 100+ 提供商），3 种模式：stable、balanced、diverse
- **模式层**：Pydantic v2（CritiqueSchema、ConsensusSchema、Config）
- **评估层**：DebateEval 7 个指标
- **输出层**：Markdown、SARIF、JSON

### 2.2 核心算法
- **CIS 计算方法**：原创的 Conformity Impact Score，量化智能体立场变化的质量
- **多智能体交互流程**：单轮和多轮辩论的完整流程
- **反从众防御机制**：三层防御确保独立思考

## 7. 项目演示

### 7.1 快速演示
- **Python API**：展示如何使用 QuickCritiqueEngine 进行单轮批判
- **REST API**：展示如何通过 HTTP 请求使用 DebateEngine
- **GitHub Action**：展示如何在 PR 中自动运行 DebateEngine

### 7.2 演示脚本
```python
import asyncio
from debate_engine import QuickCritiqueEngine
from debate_engine.schemas import CritiqueConfigSchema, TaskType

async def main():
    engine = QuickCritiqueEngine()
    
    config = CritiqueConfigSchema(
        content='''def login(username, password):
    query = f"SELECT * FROM users WHERE name='{username}' AND pass='{password}'"
    return db.execute(query)''',
        task_type=TaskType.CODE_REVIEW,
    )
    
    consensus = await engine.critique(config)
    print(consensus.final_conclusion)
    print(f"Confidence: {consensus.consensus_confidence}")
    for critique in consensus.critiques_summary:
        print(f"  [{critique.severity.value}] {critique.target_area}: {critique.evidence[:100]}")

asyncio.run(main())
```
